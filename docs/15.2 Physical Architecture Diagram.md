## Key as-built architecture note

The most important current-state architecture detail is that WiseCoach is **not** routed through Cloud Functions. The current codebase implements a **direct client-to-OpenAI** path from the Flutter app:

- `lib/screens/coach/coach_screen.dart`
- `lib/screens/plans/post_session_summary_screen.dart`
- `.env`

For the report, that means:

- draw **Flutter app -> OpenAI API** as a **solid implemented connection**
- draw **Flutter app -> Cloud Functions -> OpenAI API** as a **dashed planned connection**
